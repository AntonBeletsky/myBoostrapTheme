Назначение файлов _variables.scss и _bootswatch.scss
Эти два файла являются ключевыми для кастомизации и создания тем для фреймворка Bootstrap с использованием препроцессора SASS.

1. _variables.scss
Это файл переменных. В контексте Bootswatch, этот файл содержит все переменные Bootstrap, которые были переопределены для создания конкретной темы.

Что внутри? Внутри вы найдете SASS-переменные, которые определяют всю визуальную составляющую Bootstrap:

Цвета ($primary, $secondary, $success, $body-bg, $body-color и т.д.)
Шрифты ($font-family-sans-serif, $font-size-base, $headings-font-weight и т.д.)
Отступы ($spacer, $grid-gutter-width)
Радиусы скругления углов ($border-radius)
Тени, переходы и многое другое.
Зачем он нужен? Вместо того чтобы переписывать тысячи строк CSS-кода Bootstrap, вы просто меняете значения в этом файле. Например, чтобы изменить основной цвет сайта с синего на фиолетовый, достаточно изменить одну переменную $primary: #some-purple-color;. Это сердце любой темы Bootswatch.

2. _bootswatch.scss
Это главный сборочный файл темы Bootswatch. Его задача — собрать все части воедино в правильном порядке.

Что внутри? Этот файл обычно делает две вещи:

Импортирует (подключает) только что упомянутый файл с переменными _variables.scss.
Импортирует всю библиотеку Bootstrap из исходников.
Иногда он также может содержать небольшое количество дополнительных стилей, уникальных для данной темы, которые нельзя реализовать через переменные.

Зачем он нужен? Он гарантирует, что ваши кастомные переменные (из _variables.scss) будут загружены до того, как загрузятся стандартные компоненты Bootstrap. Таким образом, когда Bootstrap начнет генерировать CSS для кнопок, алертов, навигации и т.д., он уже будет использовать ваши новые цвета, шрифты и настройки, а не стандартные.

Итог: _variables.scss — это "что" меняем (палитра, шрифты), а _bootswatch.scss — это "как" собираем (инструкция по сборке).

Теперь самое главное. Создайте файл custom.scss в папке scss/. Этот файл будет точкой входа для компилятора.
